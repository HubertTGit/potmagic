# i18n Design — English + German

**Date:** 2026-03-24
**Status:** Approved
**Scope:** Full app (landing pages + authenticated app)

---

## Summary

Add English/German internationalisation to potmagic using `react-i18next` with TypeScript module augmentation for type-safe translation keys. Language is encoded in the URL (`/de/` prefix for German, no prefix for English) using TanStack Router's optional `($lang)` segment. A language switcher button appears in the landing navbar and the authenticated sidebar, next to the existing theme toggle.

---

## Decisions

| Question | Decision | Rationale |
|---|---|---|
| Library | `react-i18next` | Battle-tested, SSR-friendly, type-safe via module augmentation |
| Language storage | URL prefix | SEO-friendly, shareable links, no hydration mismatch |
| English URL | No prefix (`/stories`) | English is default; `/de/stories` for German |
| Type-safety | TypeScript module augmentation | Zero extra tooling; keys typed against `en.json` |
| Namespace strategy | Single flat namespace with feature prefixes | Simple; two languages don't warrant multi-namespace |

## 1. Routing Architecture

### Optional `($lang)` segment

All routes move into a `($lang)/` folder under `src/routes/`. TanStack Router's optional param syntax means a single route tree handles both `/concept` and `/de/concept`.

**Target file structure:**

```
src/routes/
  __root.tsx                          (unchanged)
  ($lang).tsx                         ← layout: validates lang, sets i18next language
  ($lang)/
    index.tsx                         →  /  and  /de/
    concept.tsx                       →  /concept  and  /de/concept
    pricing.tsx
    docs.tsx
    docs/
      index.tsx
      create-story.tsx
      add-scenes.tsx
      props.tsx
      size-guidelines.tsx
    auth.tsx
    auth.director-setup.tsx
    under-construction.tsx
    show/
      index.tsx
      $storyId.tsx
    _app.tsx
    _app/
      profile.tsx
      director.tsx
      stories/
        index.tsx
        $storyId/
          index.tsx
          scenes/$sceneId.tsx
      stage/
        index.tsx
        $sceneId.tsx
```

### `($lang).tsx` layout route

```ts
// src/routes/($lang).tsx
import { createFileRoute, redirect, Outlet } from '@tanstack/react-router'
import i18n from '@/i18n'

export const Route = createFileRoute('/($lang)')({
  beforeLoad: ({ params }) => {
    const lang = params.lang ?? 'en'
    if (lang !== 'en' && lang !== 'de') {
      throw redirect({ to: '/' })
    }
    i18n.changeLanguage(lang)
  },
  component: function LangLayout() {
    const { lang } = Route.useParams()
    useEffect(() => {
      document.documentElement.lang = lang ?? 'en'
    }, [lang])
    return <Outlet />
  },
})
```

Validation rejects any unknown `lang` value (e.g. `/fr/`, `/xyz/`) with a redirect to `/`.

## 2. i18n Library Setup

### Packages

```
pnpm add react-i18next i18next
```

### File structure

```
src/
  i18n/
    en.json         ← source of truth; TypeScript typed against this
    de.json         ← German translations (same keys)
    index.ts        ← i18next initialisation
    i18next.d.ts    ← TypeScript module augmentation
```

### Translation key convention

Flat keys with a feature prefix. No nested objects — keeps JSON diffing clean.

```json
// src/i18n/en.json (excerpt)
{
  "nav.pricing": "Pricing",
  "nav.concept": "Concept",
  "nav.docs": "Docs",
  "nav.overview": "Overview",
  "nav.joinTheatre": "Join Theatre",
  "nav.startCurating": "Start Curating",
  "nav.watchLive": "Watch Live",
  "nav.stories": "Stories",
  "nav.stage": "Stage",
  "nav.director": "Director",
  "nav.profile": "Profile",

  "hero.headline": "Live",
  "hero.subline": "Whenever, wherever.",
  "hero.cta": "Join Our Theatre",

  "concept.vision": "The Vision",
  "concept.headline": "Live Story Telling",

  "auth.joinTheatre": "Join Theatre",
  "auth.signIn": "Sign in"
}
```

### Type-safety

```ts
// src/i18n/i18next.d.ts
import en from './en.json'

declare module 'i18next' {
  interface CustomTypeOptions {
    defaultNS: 'translation'
    resources: {
      translation: typeof en
    }
  }
}
```

After this, `t('nav.pricig')` is a TypeScript compile error. Editor auto-complete works on all keys.

### Initialisation

```ts
// src/i18n/index.ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import en from './en.json'
import de from './de.json'

i18n.use(initReactI18next).init({
  lng: 'en',
  fallbackLng: 'en',
  resources: {
    en: { translation: en },
    de: { translation: de },
  },
  interpolation: { escapeValue: false },
})

export default i18n
```

Imported once at the app entry point (TanStack Start client bootstrap).

### SSR / hydration

Language is derived entirely from the URL. The `beforeLoad` on `($lang).tsx` calls `i18n.changeLanguage()` synchronously on both server and client — no hydration mismatch is possible. No `localStorage` is involved for language (only theme uses `localStorage`).

## 3. `useLanguage` Hook

A single hook for all components. Avoids direct `react-i18next` imports in component files.

```ts
// src/hooks/useLanguage.ts
import { useTranslation } from 'react-i18next'
import { useParams } from '@tanstack/react-router'

export function useLanguage() {
  const { t } = useTranslation()
  const { lang } = useParams({ strict: false })
  const locale = lang ?? 'en'
  const langPrefix = locale === 'de' ? '/de' : ''

  return { t, locale, langPrefix }
}
```

Usage in components:

```tsx
const { t, langPrefix } = useLanguage()
<Link to={`${langPrefix}/stories`}>{t('nav.stories')}</Link>
```

## 4. Language Switcher Component

```tsx
// src/components/language-switcher.component.tsx
import { useNavigate, useRouterState } from '@tanstack/react-router'
import { useLanguage } from '@/hooks/useLanguage'

export function LanguageSwitcher() {
  const { locale } = useLanguage()
  const navigate = useNavigate()
  const pathname = useRouterState({ select: (s) => s.location.pathname })

  const toggle = () => {
    if (locale === 'en') {
      navigate({ to: `/de${pathname}` })
    } else {
      navigate({ to: pathname.replace(/^\/de/, '') || '/' })
    }
  }

  return (
    <button
      type="button"
      onClick={toggle}
      className="btn btn-ghost btn-sm btn-square font-display text-xs tracking-wider"
      aria-label="Switch language"
    >
      {locale === 'en' ? 'DE' : 'EN'}
    </button>
  )
}
```

**Button label convention:** shows the *target* language (what you switch *to*), matching the sun/moon theme toggle.

### Placement

- **`landing-navbar.component.tsx`** — immediately left of the theme toggle button, in both desktop and mobile menus
- **`sidebar.component.tsx`** — next to the theme toggle at the bottom of the sidebar

### Edge case behaviour

| Current URL | Switch to | Result |
|---|---|---|
| `/` | DE | `/de/` |
| `/concept` | DE | `/de/concept` |
| `/stories/abc123` | DE | `/de/stories/abc123` |
| `/de/stage/xyz` | EN | `/stage/xyz` |
| `/de/` | EN | `/` |

## 5. Migration Steps (high-level)

1. Install `react-i18next` and `i18next`
2. Create `src/i18n/` with `en.json`, `de.json`, `index.ts`, `i18next.d.ts`
3. Import `src/i18n/index.ts` in the app entry point
4. Move all route files into `src/routes/($lang)/`
5. Create `src/routes/($lang).tsx` layout
6. Create `src/hooks/useLanguage.ts`
7. Create `src/components/language-switcher.component.tsx`
8. Update `landing-navbar.component.tsx` — add `<LanguageSwitcher />`, update all `<Link>` `to` props to use `langPrefix`, replace hardcoded strings with `t()`
9. Update `sidebar.component.tsx` — add `<LanguageSwitcher />`, update links and strings
10. Translate all hardcoded strings in landing pages and authenticated UI into `en.json` / `de.json`
11. Update `__root.tsx` `RootDocument` to set `lang` attribute from route context
