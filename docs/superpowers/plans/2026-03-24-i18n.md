# i18n (English + German) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add English/German internationalisation using `react-i18next`, an optional `($lang)` URL prefix, and a language switcher in the navbar and sidebar.

**Architecture:** All routes (except `show/`) move into a `($lang)/` folder — TanStack Router's optional param means one route tree handles both `/concept` and `/de/concept`. Language is detected from the URL, not localStorage. i18next is initialised once at the client entry point and switched client-side in a `useEffect` to avoid the SSR singleton problem.

**Tech Stack:** `react-i18next`, `i18next`, TanStack Router file-based routing (`($lang)` optional segment), React 19, TanStack Start (SSR), TypeScript module augmentation for key type-safety.

**Spec:** `docs/superpowers/specs/2026-03-24-i18n-design.md`

**Branch note:** Do all of this on a dedicated branch (`intl` already exists). Task 5 (moving route files) breaks the app until Task 6 (route ID updates) is complete — do not commit them separately.

## Task 2: Create `useLanguage` hook

**Files:**
- Create: `src/hooks/useLanguage.ts`

- [ ] **Step 1: Create the hook**

Create `src/hooks/useLanguage.ts`:

```ts
import { useTranslation } from 'react-i18next'
import { useParams } from '@tanstack/react-router'

/**
 * Provides translation function and locale utilities.
 *
 * Reads `lang` param with `strict: false` — works in any route in the tree.
 * On routes outside the `($lang)` subtree (e.g. `show/`), `locale` is always `'en'`.
 */
export function useLanguage() {
  const { t } = useTranslation()
  const { lang } = useParams({ strict: false })
  const locale = lang ?? 'en'
  const langPrefix = locale === 'de' ? '/de' : ''

  return { t, locale, langPrefix }
}
```

## Task 3: Create `($lang).tsx` layout route

**Files:**
- Create: `src/routes/($lang).tsx`

- [ ] **Step 1: Create the layout**

Create `src/routes/($lang).tsx`:

```tsx
import { createFileRoute, redirect, Outlet } from '@tanstack/react-router'
import { useEffect } from 'react'
import i18n from '@/i18n'

export const Route = createFileRoute('/($lang)')({
  beforeLoad: ({ params }) => {
    const lang = params.lang
    // undefined = English (no prefix), 'de' = German
    // Any other value is invalid — redirect to home
    if (lang !== undefined && lang !== 'de') {
      throw redirect({ to: '/' })
    }
    return { locale: lang ?? 'en' }
  },
  component: function LangLayout() {
    const { lang } = Route.useParams()
    const locale = lang ?? 'en'

    useEffect(() => {
      // client-only: avoids SSR singleton mutation on concurrent requests
      i18n.changeLanguage(locale)
      document.documentElement.lang = locale
    }, [locale])

    return <Outlet />
  },
})
```

## Task 4: Create `LanguageSwitcher` component

**Files:**
- Create: `src/components/language-switcher.component.tsx`

This task is done before the route migration (Task 5) so the component is ready to drop in.

- [ ] **Step 1: Create the component**

Create `src/components/language-switcher.component.tsx`:

```tsx
import { useNavigate, useRouterState } from '@tanstack/react-router'
import { useLanguage } from '@/hooks/useLanguage'

/**
 * Toggles between EN (no prefix) and DE (/de/ prefix).
 * Button shows the TARGET language — matching sun/moon theme toggle convention.
 * Only used in navbar and sidebar — never rendered on show/ routes.
 */
export function LanguageSwitcher() {
  const { locale } = useLanguage()
  const navigate = useNavigate()
  const pathname = useRouterState({ select: (s) => s.location.pathname })

  const toggle = () => {
    if (locale === 'en') {
      navigate({ to: `/de${pathname}` })
    } else {
      const stripped = pathname.replace(/^\/de/, '') || '/'
      navigate({ to: stripped })
    }
  }

  return (
    <button
      type="button"
      onClick={toggle}
      className="btn btn-ghost btn-sm btn-square font-display text-xs tracking-wider"
      aria-label="Switch language"
    >
      {locale === 'en' ? 'DE' : 'EN'}
    </button>
  )
}
```
